# Customer Journey

![VayuNetrix customer workflow](../../diagrams/vayunetrix-customer-workflow.svg)

## Journey Steps

1. Register
2. Confirm email
3. Select package
4. Connect workspace
5. Validate intake
6. Generate platform plan
7. Review plan
8. Approve changes
9. Provision through guarded workflows
10. Review evidence
11. Operate and govern continuously

## Workspace Experience

After login, the customer sees a workspace home that summarizes:

- selected package
- AWS setup progress
- current risk status
- pending approvals
- latest run status
- cost estimate
- evidence and report status

## Guided Architecture Builder

The architecture builder asks guided questions about:

- business size
- regions and data residency
- environments
- account model
- governance level
- security baseline
- networking pattern
- cost controls

The goal is to guide customers instead of expecting them to already know every AWS landing-zone decision.

## Approval-Centric Operations

Before high-impact actions, the platform shows:

- what will change
- affected accounts, OUs, roots, regions, and services
- security impact
- cost impact
- blast-radius summary
- required approvals

This keeps provisioning and remediation deliberate rather than automatic and opaque.
